# CLAUDE.md の保守不足による情報の陳腐化

## 概要

`CLAUDE.md` に記載されている情報が実態と乖離しており、PR #165 のレビューで Copilot に複数指摘された。CLAUDE.md は Claude Code や貢献者が最初に参照するファイルであるため、誤情報が混入すると開発体験に直接影響する。

## 詳細

発覚した具体的な誤記:

| 箇所 | 誤記 | 正しい内容 |
|---|---|---|
| プロジェクト概要 | `Go 1.24+` | `go.mod` の `go 1.26.1` |
| ビルドコマンド例 | `make dev ARGS="pull --store --users"` | `--store` フラグは存在しない。正: `pull --users` |

Go バージョンは `go.mod` が更新されたタイミングで CLAUDE.md が追随できていなかった。CLI フラグについても、コマンド仕様変更時に CLAUDE.md を更新するフローがなかったことが原因と思われる。

## 提案

- `go.mod` の Go バージョン変更時に CLAUDE.md を同一 PR で更新することをルール化する
- CLAUDE.md に記載するコマンド例は `make run-help` 等の実際の出力から機械的に確認できるものに限定する
- 定期的（月次など）に CLAUDE.md の記載内容と実装を突き合わせてレビューする
